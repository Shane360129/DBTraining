| 改動 | v0321（舊） | v0322（新） |
|------|-------------|-------------|
| **格式** | 純文字 `Table:/.../SQL:` | Llama-3.1 Chat Template（system/user/assistant） |
| **Schema** | ❌ 不提供 | ✅ 單表 CREATE TABLE（含欄位型態） |

4. **子查詢去重模式**：Rules 中明確教導 `SELECT SUM(amount) FROM (SELECT DISTINCT ...) sub`
5. **移除 v3 雜訊**：只用 spider + claude_en（0317 最佳組合）

## 七、關鍵發現總結

### 7.1 每個 Phase 學到的教訓

| Phase | 教訓 |
|-------|------|
| **Phase 1** | DoRA 比 LoRA 更穩定；LR 3e-4 太高，需降至 1e-4 |
| **Phase 2** | 精簡高品質資料(700) > 大量低品質(1,322)；修正樣本效果顯著(+10% per batch) |
| **Phase 3** | 91.76% EM 是假象（66.5% 洩漏）；v3 中文資料品質差，引入雜訊反而退步 |
| **Phase 4** | 純文字格式是瓶頸；需 Schema + 業務規則 + Chat Template 才能真正提升 |

### 7.2 EM 進步與退步的原因

| 變化 | 原因 |
|------|------|
| 34% → 52% (v0307→v0313) | 精簡資料 + 移除方括號格式 |
| 52% → 57% (v0313→v0314) | +90 筆新資料 |
| 57% → 67% (v0314→v0315) | +82 筆修正樣本（針對性修正） |
| 67% → 92% (v0315→v0317) | +103 筆修正樣本 + Auto Loop（但含洩漏） |
| 44% → 26% (0317→v3 only) | v3 資料品質差，COUNT 偏重，無子查詢 |
| 44% → 41% (0317→v0321 合併) | v3 雜訊拖累，更多資料≠更好 |

### 7.3 目前 5 大瓶頸（v0322 嘗試解決）

| # | 瓶頸 | 影響 | v0322 解法 |
|---|------|------|-----------|
| 1 | WP_vOutStock 全面失敗 (5.9%) | 34 筆 (14.3%) | Schema 中列出所有表 + 欄位 |
| 2 | 子查詢去重 0% 成功 | 28 筆 (11.8%) | Rules 中明確教導子查詢模式 |
| 3 | isDel/dtlIsDel 混淆 | 75 筆 (31.5%) | 表專屬 Rules 明確區分 |
| 4 | 欄位名稱錯誤 | 20 筆 (8.4%) | CREATE TABLE 提供正確欄位名 |
| 5 | 訓練/推論格式不一致 | 全部樣本 | 統一 Chat Template |
