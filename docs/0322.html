歷史評估結果與過擬合現象 Phase 1 to Phase 3

v0320 評估顯示，前期 91.76% 的超高 EM 源自於舊測試集的資料洩漏（Data Leakage）。在嚴格的新驗證集下，EM 暴跌但 EX 守在 44.12%，這將作為我們真正的基準起點。

資料庫分析與 Schema 設計 WP_M09 環境

商業邏輯提取與壓縮

  • 架構分析： 完整解析 SQL Server 7 個核心 View 之關聯與邏輯。
  • Schema 壓縮技術： 將原始 CREATE TABLE 語法精簡為緊湊欄位列表，大幅節省 Context Window。
  • 自主選表能力： 放棄單表提示，注入完整 7 表 Schema，強制模型學習根據語意挑選正確的 Table/View。

isDel 軟刪除機制

4 個交易視圖具備軟刪除，必須強制加入 isDel='N'。

ID 日期編碼規則

單據 ID 前 8 碼為日期，需用 LEFT(Id, 8) 過濾。

pNo 非日期陷阱

pNo 為流水號，嚴禁模型誤判為日期進行比對。
